# Инструкция по развертыванию

## 1. Развертывание Mini App на GitHub Pages

1. Создайте репозиторий `tohatchbot/hatch` на GitHub
2. Загрузите файлы из папки `hatch_miniapp` в корень репозитория
3. В настройках репозитория включите GitHub Pages
4. Выберите ветку `main` как источник
5. Mini app будет доступна по адресу: `https://tohatchbot.github.io/hatch/`

## 2. Настройка API Endpoint

Mini app требует API endpoint для получения статистики. Есть несколько вариантов:

### Вариант 1: Использовать бота как API сервер

Бот уже настроен для работы как API сервер на порту 8080. Нужно:
1. Настроить reverse proxy (nginx) для доступа к API
2. Или использовать сервис типа Cloudflare Tunnel
3. Обновить `API_URL` в `app.js` на ваш домен

### Вариант 2: Использовать отдельный сервер

Создайте отдельный веб-сервер, который будет получать данные из бота через:
- Базу данных (если бот использует БД)
- Redis
- Или другой механизм обмена данными

### Вариант 3: Использовать CORS proxy

Для тестирования можно использовать CORS proxy, но для продакшена это не рекомендуется.

## 3. Обновление URL в боте

После развертывания mini app обновите URL в `bot.py`:
- Убедитесь, что URL указывает на `https://tohatchbot.github.io/hatch/`

## 4. Настройка CORS

Если API сервер находится на другом домене, убедитесь что:
- CORS headers настроены правильно (уже добавлены в код)
- API доступен публично или через аутентификацию
